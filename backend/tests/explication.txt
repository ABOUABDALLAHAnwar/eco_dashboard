1️⃣ Tests unitaires (Unit Tests)

Objectif : tester les composants isolés, sans dépendances externes.

Schéma de test :

+---------------------+
| CollectionCrud      |
|---------------------|
| - create(user)      |
| - read(email)       |
| - update(email, d)  |
| - delete(email)     |
+---------------------+
           |
           v
  Test unitaire pour chaque méthode :
  - create : insère un utilisateur
  - read : récupère l’utilisateur
  - update : modifie un utilisateur existant
  - delete : supprime un utilisateur


Points clés à tester :

Les entrées valides → le résultat attendu est correct

Les entrées invalides → l’exception est levée

Les cas limites → email déjà existant, utilisateur absent
